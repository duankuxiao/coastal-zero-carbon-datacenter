# 未来数据中心增长的沿海布局：海水冷却、海上风电与全球零碳算力潜力

**Proposed title:**  
**Coastal siting of future data center growth: seawater cooling, offshore wind, and the global potential for zero-carbon computing**

---

## 研究定位

本文面向 *Nature Energy*、*Joule* 等能源系统类期刊，聚焦全球数据中心快速增长背景下，新增算力基础设施的空间布局、冷却方式和零碳供电路径。本文将数据中心按照**是否位于沿海城市**划分为两类：

- **沿海数据中心**：位于距离海岸距离为 0 的城市，可在情景中假设具备使用海水源热泵或海水自然冷却的潜力，并可优先匹配海上风电资源。
- **非沿海数据中心**：不具备直接利用海水冷源和近海海上风电直供条件的数据中心，主要依赖常规冷却和区域电网供电。

本文的核心问题是：

> 在未来全球和中国数据中心持续增长的背景下，如果部分新增容量被布局在沿海城市，并通过海水冷却降低冷却能耗、通过海上风电提供零碳电力，该路径能否支撑数据中心实现年度绿电平衡甚至小时级零碳运行？如果不能满足，还存在多大绿电缺口，以及需要通过何种方式补足？

由于海上光伏仍处于初步发展阶段，当前容量较小、工程和成本不确定性较高，本文不将海上光伏纳入主要供电选项，只考虑海上风电作为沿海数据中心的主要近海零碳电源。

---

## 核心研究问题

本文拟回答以下问题：

1. 未来全球及中国数据中心新增容量可能增长到什么规模？
2. 未来新增数据中心中，沿海城市能够承接多少比例的容量？
3. 沿海数据中心相较于非沿海数据中心，能否通过海水源热泵或海水自然冷却降低冷却能耗？
4. 海上风电在年度电量尺度上能否满足沿海数据中心的用电需求？
5. 在小时级供需匹配约束下，海上风电能否支撑沿海数据中心实现 24/7 零碳运行？
6. 如果现有海上风电增长趋势无法满足数据中心需求，绿电缺口有多大？
7. 这些缺口应通过海上风电加速开发、跨区绿电交易、储能、蓄冷、数据中心负荷调度或稳定零碳电源中的哪些方式补足？
8. 该路径对全球沿海地区数据中心规划、海上风电开发、电力系统灵活性和零碳算力政策有何启示？

---

## 论文主线假设

全文建立在四个核心假设上：

1. **未来新增数据中心容量具有一定空间可塑性。** 并非所有算力负荷都必须在现有数据中心集群中增长，部分低时延需求、边缘推理、港口工业互联网、金融服务、城市云服务和区域 AI 应用负荷可以在沿海城市部署。
2. **沿海数据中心具备冷却侧优势。** 与普通空气源或常规冷却系统相比，海水冷源可在部分气候区降低冷却系统能耗，并在低温海水条件下支持自然冷却。
3. **沿海数据中心具备供电侧协同潜力。** 海上风电靠近沿海负荷中心，具有为沿海数据中心提供低碳电力的空间协同优势。
4. **年度绿电平衡不等于小时级零碳。** 即使年度发电量能够覆盖数据中心用电，仍需进一步评估小时级风电出力与数据中心连续负荷之间的时间错配。

# 1. 背景

## 1.1 全球数据中心用电增长与能源系统压力

本节说明数据中心为何从数字基础设施问题转变为能源系统问题。

论述要点：

- AI、云计算、边缘计算和数字经济推动数据中心需求快速增长；
- 数据中心从过去的局部用电设施，逐渐成为影响区域电力规划、可再生能源消纳和电网投资的重要负荷；
- 数据中心负荷具有高连续性、高可靠性和高功率密度特征；
- 传统数据中心布局通常围绕土地、电价、网络连接、气候条件、税收政策和电网接入展开；
- 未来数据中心布局必须进一步考虑低碳电力可获得性、冷却资源、电网调节能力和区域碳约束；
- 数据中心绿色化不能只关注 PUE，还应关注电力来源、小时级无碳电力匹配率和全生命周期碳排放。

## 1.2 现有数据中心低碳路径的局限

传统数据中心减碳路径包括：

- 提高服务器和 IT 设备能效；
- 提高冷却系统效率；
- 利用自然冷却；
- 购买绿电或绿证；
- 签订可再生能源 PPA；
- 配置储能或参与需求响应；
- 通过区域调度提升算力负荷灵活性。

但这些路径仍存在不足：

- 内陆地区可能具备低电价或可再生能源基地优势，但未必接近低时延需求中心；
- 温暖地区数据中心冷却电耗较高，单纯提高 IT 设备效率难以解决全部能耗问题；
- 年度绿电采购不能保证小时级零碳；
- 风电出力与数据中心连续负荷之间存在时间错配；
- 现有研究较少将新增数据中心空间布局、海水冷却、海上风电和 24/7 零碳需求放在统一框架下分析。

## 1.3 本文提出的替代视角

本文提出一个系统性问题：

> 如果未来新增数据中心容量的一部分被布局在沿海城市，利用海水冷源降低冷却需求，并通过海上风电和灵活性资源提供低碳电力，其节能减排和零碳潜力有多大？

本文贡献不是单一技术创新，而是将**空间布局、冷却技术、海上风电、电力系统灵活性和政策机制**结合起来，评估未来新增算力的零碳路径。

# 2. 数据中心沿海与非沿海增长情景

本节为需求侧情景设计，核心是估计未来数据中心新增容量，并判断其中有多少容量可能分布在沿海城市。

## 2.1 沿海与非沿海数据中心划分

本文不再按照国家类型划分分析对象，而是按照数据中心所在地是否具备沿海条件进行划分。

| 类型      | 判定标准            | 技术假设                      | 主要分析目的               |
| ------- | --------------- | ------------------------- | -------------------- |
| 沿海数据中心  | 位于距离海岸距离为 0 的城市 | 可使用海水源热泵或海水自然冷却，可优先匹配海上风电 | 评估海水冷却节能和海上风电零碳供电潜力  |
| 非沿海数据中心 | 不满足上述条件         | 主要采用常规冷却方式，依赖区域电网和陆上绿电    | 作为对比基准，反映惯性布局下的能耗和排放 |

沿海数据中心并不意味着所有负荷都必须迁移到沿海，而是指未来新增容量中具备空间调整可能的部分，可以优先布局在具备冷却和海上风电协同潜力的沿海城市。

## 2.2 数据中心增长情景

本节将原先的数据中心增长情景调整为三类，重点不是单纯设置全国总量的高、中、低增长，而是讨论**沿海数据中心容量占比相对于现状比例的变化**。

设当前沿海城市数据中心容量占全国或全球数据中心总容量的比例为：

$[p_{coast,0}]$

则未来沿海数据中心容量占比设置为：

| 情景          | 沿海数据中心容量占比设定             | 含义         | 解释                                                  |
| ----------- | ------------------------:| ---------- | --------------------------------------------------- |
| S1 沿海受限增长情景 | $0.8 \times p_{coast,0}$ | 现状比例的 80%  | 沿海城市用电需求趋于饱和，新增电力负荷增长空间有限，因此未来数据中心容量占比低于现状          |
| S2 现状比例延续情景 | $1.0 \times p_{coast,0}$ | 现状比例的 100% | 数据中心按照当前空间格局继续增长，沿海与非沿海容量比例基本不变                     |
| S3 沿海增强增长情景 | $1.2 \times p_{coast,0}$ | 现状比例的 120% | AI 需求增加，且沿海地区具备较好的冷却条件、海上风电资源和网络条件，因此未来数据中心容量占比高于现状 |

需要说明的是，三类情景中的 80%、100% 和 120% 是对**沿海数据中心容量占比**的调整，而不是直接表示全国数据中心总容量增长率。总容量增长可由外部数据中心需求预测决定，空间分布比例则由上述三类情景决定。

# 3. 海水源热泵与常规冷却系统的差异及节能潜力

本节聚焦冷却侧，说明沿海数据中心为什么可能具有能源效率优势。

## 3.1 数据中心冷却的基本问题

数据中心总用电主要包括：

- IT 设备用电；
- 冷却系统用电；
- 配电和 UPS 损耗；
- 风机、水泵、照明、消防、监控等辅助用电。

冷却系统节能的核心在于降低：

- 压缩机制冷能耗；
- 冷却塔或风冷系统能耗；
- 水泵和风机能耗；
- 高温季节制冷效率下降导致的额外电耗。

## 3.2 常规冷却系统的局限

常规冷却路径通常包括：

- 风冷冷水机组；
- 水冷冷水机组与冷却塔；
- 机械制冷为主的全年运行模式；
- 以空气或冷却塔为主要散热对象的冷却系统。

其主要局限包括：

- 夏季高温时制冷效率下降；
- 空气源冷却受环境气温影响明显；
- 冷却塔可能消耗大量淡水；
- 高密度 AI 机柜增加传统冷却系统压力；
- 在水资源紧张地区，WUE 可能成为约束；
- 当数据中心持续满负荷运行时，冷却系统难以通过短时调度显著降低能耗。

## 3.3 海水源热泵和海水自然冷却的优势

海水源热泵的核心优势是利用海水作为相对稳定的大规模冷源。

潜在优势包括：

- 海水温度波动通常小于空气温度；
- 在部分季节可提高制冷效率；
- 低温海水条件下可实现自然冷却；
- 可减少或替代冷却塔淡水消耗；
- 适合沿海大型园区级数据中心冷却系统；
- 与港口、沿海工业园区、海洋能源和低碳算力园区存在协同空间。

本节需要区分两类节能机制：

| 机制     | 含义                     | 主要影响       |
| ------ | ---------------------- | ---------- |
| 海水源热泵  | 利用海水作为热泵冷源或热汇，提高制冷系统效率 | 降低机械制冷电耗   |
| 海水自然冷却 | 当海水温度足够低时，减少或绕开压缩机制冷   | 显著降低冷却系统电耗 |

## 3.4 工程约束与环境约束

需要主动讨论海水冷却的限制条件：

- 海水温度并非总是低，热带和亚热带海域节能效果可能有限；
- 取排水距离和水泵能耗可能抵消部分收益；
- 腐蚀、结垢、生物附着会增加维护成本；
- 排海温升可能影响近岸生态；
- 台风、风暴潮和海平面上升会影响系统可靠性；
- 海域使用权、环保审批和沿海土地资源可能成为政策障碍。

因此，海水冷却的节能潜力高度依赖海水温度、取排水工程、换热效率、设备防腐、污堵控制和数据中心冷冻水温度设定。本文应避免将海水冷却描述为对所有沿海城市都有效的通用方案，而应将其作为受气候、工程和监管条件约束的区域性低碳路径。

## 3.5 需求侧计算指标

每个情景需要输出以下需求侧指标：

| 指标         | 解释                               |
| ---------- | -------------------------------- |
| 未来数据中心总容量  | 全球或中国未来数据中心 IT 容量或用电规模           |
| 沿海数据中心容量   | 按照第二节 S1–S3 情景分配至沿海城市的数据中心容量     |
| 非沿海数据中心容量  | 总容量扣除沿海容量后的剩余部分                  |
| 沿海数据中心总用电量 | 考虑 IT 负荷、冷却系统和辅助系统后的总用电需求        |
| 冷却电耗变化     | 海水冷却相较于常规冷却带来的电耗降低               |
| 年度零碳电力需求   | 沿海数据中心实现年度绿电覆盖所需的海上风电电量          |
| 小时级零碳电力需求  | 沿海数据中心实现 24/7 零碳所需的小时级无碳电力与灵活性资源 |

# 4. 海上风电供给、绿电缺口与小时级零碳潜力

本节是全文的核心结果框架。从需求侧反推：为了满足沿海数据中心用电需求，需要多少海上风电供给比例；现有海上风电增长趋势是否能够满足该需求；如果不能满足，还存在多大绿电缺口，以及需要通过何种方式补足。

## 4.1 供需反推框架

本节的逻辑顺序如下：

1. 计算不同数据中心增长情景下沿海数据中心的年度用电需求；
2. 根据海水冷却节能效果修正沿海数据中心总用电；
3. 计算实现年度 100% 绿电覆盖所需的海上风电电量；
4. 将该电量换算为所需海上风电装机容量或所需海上风电增长比例；
5. 与现有海上风电增长趋势、规划装机和可开发潜力对比；
6. 判断现有增长趋势是否足以满足沿海数据中心需求；
7. 如果年度电量足够，进一步评估小时级零碳可行性；
8. 如果年度电量不足，计算绿电缺口并讨论补足路径。

## 4.2 海上风电增长趋势与可供给能力

海上风电供给侧需要设置以下信息：

| 指标           | 解释                                 |
| ------------ | ---------------------------------- |
| 现有海上风电装机     | 当前可用于评估的基础装机规模                     |
| 未来海上风电增长趋势   | 按现有政策、产业趋势和规划推算的增长路径               |
| 海上风电容量因子     | 将装机容量转化为年度发电量                      |
| 小时级出力曲线      | 用于评估与数据中心负荷之间的时间匹配                 |
| 可接入沿海数据中心的电量 | 通过直供、绿电交易、PPA、源网荷储一体化等机制与数据中心匹配的电量 |
| 空间约束         | 海域开发、并网、送出、生态和审批约束                 |

需要强调的是，海上风电并非天然属于数据中心。数据中心能否使用海上风电，取决于电力市场机制、绿电交易制度、物理电网约束和小时级零碳认证规则。

## 4.3 年度绿电覆盖率

年度绿电覆盖率用于判断在年电量尺度上，海上风电能否覆盖沿海数据中心用电需求。

可定义为：

$[
R_{annual}=\frac{E_{offshore\ wind}}{E_{DC,coast}}
]$

其中，$(E_{offshore\ wind})$ 为可用于沿海数据中心的海上风电年度电量，$(E_{DC,coast})$为沿海数据中心年度总用电需求。

判断标准：

| 结果                  | 含义                    |
| ------------------- | --------------------- |
| $R_{annual} \geq 1$ | 年度海上风电电量可覆盖沿海数据中心用电需求 |
| $R_{annual} < 1$    | 年度海上风电电量不足，存在绿电缺口     |

但年度绿电覆盖率只能说明年度电量是否足够，不能说明每小时是否实现零碳。

## 4.4 小时级零碳匹配率

在年度电量足够的基础上，需要进一步评估小时级供需匹配。数据中心负荷通常较为连续，而海上风电出力具有波动性，因此年度电量充足仍可能存在小时级缺口。

小时级绿电匹配率可用于衡量每小时海上风电直接满足数据中心负荷的程度：

$[
R_{hourly}=\frac{\sum_t \min(G_t,L_t)}{\sum_t L_t}
]$

其中，$(G_t)$为第 $(t)$小时可用于数据中心的海上风电出力，$(L_t)$为第$(t)$小时沿海数据中心负荷。

需要输出：

- 无储能条件下的小时级绿电匹配率；
- 配置不同储能时长后的小时级绿电匹配率；
- 达到 80%、90%、95% 和 100% 小时级无碳匹配所需的储能容量；
- 在连续低风时段下仍无法覆盖的零碳缺口。

## 4.5 绿电缺口与零碳缺口

当现有海上风电增长趋势无法满足沿海数据中心需求时，需要区分两类缺口。

| 缺口类型    | 定义                            | 含义                     |
| ------- | ----------------------------- | ---------------------- |
| 年度绿电缺口  | 年度海上风电可供电量与沿海数据中心年度用电需求之间的差值  | 表示年电量尺度上还缺多少绿电         |
| 小时级零碳缺口 | 在小时级匹配条件下，海上风电、储能和调度后仍无法满足的负荷 | 表示实现 24/7 零碳仍缺少的无碳供给能力 |

可定义年度绿电缺口为：

$[
Gap_{annual}=\max(0,E_{DC,coast}-E_{offshore\ wind})
]$

小时级零碳缺口为：

$[
Gap_{hourly}=\sum_t \max(0,L_t-G_t-S_t)
]$

其中，$(S_t)$ 表示储能、蓄冷或其他灵活性资源在第$(t)$小时提供的有效补偿能力。

## 4.6 缺口补足方式

如果海上风电无法完全满足沿海数据中心用电需求，需要讨论以下补足方式：

| 补足方式            | 主要作用                      | 适用缺口类型              |
| --------------- | ------------------------- | ------------------- |
| 加速海上风电开发        | 增加年度绿电供给                  | 年度绿电缺口              |
| 跨区绿电交易          | 通过空间互补获得更多低碳电力            | 年度绿电缺口和部分小时级缺口      |
| 电池储能            | 解决小时级和日内错配                | 小时级缺口               |
| 蓄冷系统            | 将冷却需求转移到风电高出力时段           | 小时级缺口               |
| 数据中心负荷调度        | 将可延迟训练、批处理或推理任务转移到绿电高出力时段 | 小时级缺口               |
| 长时储能或氢能         | 应对连续低风天气                  | 多日尺度缺口              |
| 核电、水电、地热等稳定零碳电源 | 提供 24/7 无碳基荷              | 小时级和多日尺度缺口          |
| 绿证和年度 PPA       | 支撑年度绿电声明                  | 年度核算缺口，但不能单独证明小时级零碳 |

本节需要强调：

> 沿海数据中心的零碳路径不是“海上风电 + 电池”的单一路径，而是海上风电、储能、蓄冷、算力调度、绿电交易和稳定零碳电源的组合。

## 4.7 核心结果指标

本节建议输出以下结果指标：

| 指标           | 解释                             |
| ------------ | ------------------------------ |
| 沿海数据中心年度用电需求 | 考虑海水冷却节能后的沿海数据中心总用电            |
| 所需海上风电装机     | 支撑年度 100% 绿电覆盖所需的海上风电容量        |
| 所需海上风电增长比例   | 相对于现有海上风电增长趋势，数据中心需要占用或新增的风电比例 |
| 年度绿电覆盖率      | 海上风电年度发电量能覆盖多少沿海数据中心用电         |
| 小时级绿电匹配率     | 每小时海上风电可直接满足多少数据中心负荷           |
| 年度绿电缺口       | 年度尺度上仍需补足的绿色电力                 |
| 小时级零碳缺口      | 实现 24/7 零碳仍缺少的无碳电力供应能力         |
| 储能需求         | 达到不同小时级零碳匹配率所需的储能功率和容量         |
| 剩余碳排放        | 绿电、储能和调度后仍由化石电力导致的排放           |

# 5. 结论与政策建议：全球潜力、政策含义与研究局限

本节总结主要发现，讨论全球推广潜力、政策含义和研究局限。

## 5.1 预期主要发现

1. **沿海布局对未来新增数据中心容量具有边际减碳价值。** 该路径不是替代所有非沿海数据中心，而是为新增低时延、边缘计算和区域 AI 负荷提供低碳布局选择。
2. **海水冷却主要贡献节能，海上风电主要贡献零碳供电。** 两者作用不同，但可以叠加降低数据中心的总电力需求和碳排放。
3. **沿海数据中心能否实现年度零碳取决于海上风电增长速度。** 若现有海上风电增长趋势能够覆盖沿海数据中心用电，则可进一步讨论小时级零碳；若不能覆盖，则需要明确绿电缺口和补足路径。
4. **年度绿电较容易，小时级零碳更困难。** 数据中心连续负荷与海上风电波动出力之间存在时间错配，需要储能、蓄冷、算力调度和稳定零碳电源共同支撑。
5. **全球潜力高度区域异质。** 北海、中国东部沿海、美国东西海岸、日本和韩国等地区可能更适合探索海上风电与沿海数据中心协同；热带沿海地区虽然具备海水冷却条件，但海水温度较高，节能效果可能较弱。
6. **海上光伏不宜作为本文主要供电路径。** 由于当前海上光伏容量较小且仍处于初步发展阶段，本文将其排除在主分析之外，以避免夸大供给侧潜力。

## 5.2 全球潜力

全球沿海数据中心零碳潜力主要取决于三类条件：

| 条件          | 作用                |
| ----------- | ----------------- |
| 数据中心需求      | 决定沿海新增容量规模和低时延需求  |
| 海水冷却条件      | 决定冷却节能潜力          |
| 海上风电资源与电力市场 | 决定年度绿电覆盖和小时级零碳可行性 |

可以将全球沿海地区分为以下类型：

| 类型             | 特征                       | 代表区域                 | 政策含义                        |
| -------------- | ------------------------ | -------------------- | --------------------------- |
| 高需求 + 高海上风电潜力  | 数据中心需求强，海上风电资源充足         | 中国东部沿海、北海地区、英国、美国东海岸 | 优先开展海上风电直供和零碳算力园区示范         |
| 高需求 + 高制冷压力    | 数据中心需求强，但气候较热，冷却压力大      | 东南亚、中东部分沿海城市         | 需要谨慎评估海水冷却效果，并强化绿电交易和高效冷却技术 |
| 高风电潜力 + 数据需求较低 | 海上风电资源好，但本地数据中心需求有限      | 澳大利亚部分沿海、部分北欧地区      | 可探索绿色算力输出或跨区域云服务            |
| 高需求但海域和土地约束强   | 数据中心需求强，但土地、海域、并网或生态约束明显 | 新加坡、日本部分地区           | 需要区域协同供电和跨境绿电机制             |

## 5.3 政策建议

### 建议 1：鼓励沿海绿色算力园区试点

沿海国家和地区可探索：

- 海上风电直供数据中心；
- 海上风电—储能—数据中心协同规划；
- 海水冷却示范园区；
- 港口、海洋经济和 AI 推理负荷协同；
- 绿电交易与 24/7 零碳认证试点。

### 建议 2：从 PUE 管理转向综合零碳评价

未来数据中心绿色评价不应只看 PUE，还应加入：

- CUE；
- 绿电比例；
- 小时级绿电匹配率；
- 24/7 无碳电力比例；
- WUE；
- 海洋生态影响；
- 储能和灵活性贡献；
- 数据中心负荷调度能力。

### 建议 3：建立海水冷却环境监管框架

需要明确：

- 取水和排水标准；
- 排海温升限制；
- 海洋生态影响评估；
- 防腐、防污、防生物附着要求；
- 台风和极端天气安全标准；
- 全生命周期运维规范。

### 建议 4：推动数据中心参与电力系统灵活性服务

数据中心不应只是刚性负荷，也可以通过：

- 可延迟计算任务调度；
- AI 推理任务区域转移；
- 蓄冷；
- UPS 和储能系统协同；
- 需求响应；
- 与海上风电出力预测联动的负荷管理；

参与电力系统调节。

### 建议 5：建立小时级绿电核算和认证机制

年度绿电采购不能直接等同于 24/7 零碳运行。若沿海数据中心需要支撑高可信度零碳算力声明，应建立：

- 小时级绿电匹配核算；
- 数据中心用电与海上风电出力的时间对应关系；
- 储能充放电的碳属性追踪；
- 绿电交易、PPA 和绿证之间的边界规则；
- 跨区绿电传输与消费归属核算机制。

## 5.4 研究局限性

本文需要明确以下局限：

1. 沿海数据中心容量比例是情景假设，不代表确定性预测；
2. 距离海岸距离为 0 的城市只是沿海数据中心的简化判定标准，实际工程仍需考虑取排水距离、土地条件和电网接入；
3. 海水冷却节能潜力受局地海水温度、取排水工程、换热效率、防腐防污和环境监管影响较大；
4. 海上风电供给能力受海域规划、并网送出、生态审批、成本和政策支持影响；
5. 小时级零碳结果高度依赖风电小时级出力数据和数据中心负荷曲线质量；
6. 本文不考虑海上光伏，可能低估部分地区未来长期海上可再生能源潜力，但有助于避免高估当前可实现的近海供电能力；
7. 全球外推可能受各国电力市场机制、数据中心监管、海域使用政策和气候条件影响。

# 研究方法与数据框架

本部分可作为正文 Methods 或补充材料的基础。

## M1. 总体方法框架

研究方法可概括为：

> 情景分析 + 沿海/非沿海空间划分 + 冷却节能评估 + 海上风电供给评估 + 年度绿电平衡 + 小时级零碳匹配 + 绿电缺口估计 + 政策解释。

流程如下：

1. 估计未来数据中心总容量或总用电需求；
2. 根据沿海数据中心现状比例构建 S1–S3 空间布局情景；
3. 将数据中心划分为沿海和非沿海两类；
4. 对沿海数据中心估算海水冷却节能潜力；
5. 计算沿海数据中心年度用电需求和小时级负荷曲线；
6. 估计海上风电年度发电量和小时级出力；
7. 反推满足沿海数据中心用电所需的海上风电装机或供给比例；
8. 判断现有海上风电增长趋势能否满足年度绿电需求；
9. 在年度满足的情形下，进一步评估小时级零碳匹配；
10. 在年度或小时级不足的情形下，计算绿电缺口和零碳缺口；
11. 评估储能、蓄冷、负荷调度、跨区绿电交易和稳定零碳电源的补足作用；
12. 进行全球沿海地区外推和政策讨论。

## M2. 数据中心增长情景方法

数据来源可包括：

- IEA 全球数据中心用电预测；
- 中国数据中心政策和行业报告；
- 各国数据中心市场增长预测；
- 云计算和 AI 服务器增长数据；
- DataCenterMap 或其他数据中心地理分布数据；
- 电力系统规划和区域负荷数据。

关键处理步骤：

- 估计未来数据中心总用电或 IT 容量；
- 识别当前沿海城市数据中心容量占比；
- 使用 80%、100%、120% 三类比例构建未来沿海数据中心容量情景；
- 比较沿海布局和非沿海惯性布局下的电力需求、冷却能耗和碳排放。

## M3. 沿海城市识别方法

沿海城市的简化判定标准为：

> 城市到海岸距离为 0，且具备理论上海水取排水条件和海上风电接入可能性。

可使用以下数据：

- 全球城市或行政区边界数据；
- 海岸线数据；
- 数据中心位置数据；
- 城市与海岸线距离；
- 海上风电资源和规划数据；
- 电网接入和负荷中心数据。

## M4. 冷却节能评估方法

对代表性国家或中国沿海地区使用海水温度和气象数据，估算：

- 可自然冷却小时数；
- 热泵运行时段；
- 常规冷却系统 COP；
- 海水源热泵 COP；
- 冷却电耗变化；
- 年均 PUE 改善空间；
- 水泵能耗、取排水能耗和辅助能耗。

应避免只比较理想状态下的制冷效率，需要将海水取排水能耗、换热损失和防污维护成本纳入敏感性分析。

## M5. 海上风电供给评估方法

以中国为详细案例，全球做外推。

需要设置：

- 海上风电现有装机；
- 海上风电未来增长趋势；
- 海上风电容量因子；
- 海上风电小时级出力；
- 并网和送出约束；
- 与沿海数据中心的空间匹配关系；
- 通过直供、PPA、绿电交易和源网荷储一体化实现的可认定绿电量。

本文不设置固定的“海上风电分配给数据中心比例”，而是反推数据中心需要多少海上风电，并与现有增长趋势进行比较。

## M6. 年度平衡、小时级匹配与储能估计

分析分为两层。

### 年度平衡

判断海上风电年度发电量是否足以覆盖沿海数据中心年用电量。

输出包括：

- 年度绿电覆盖率；
- 所需海上风电装机；
- 现有增长趋势下的可满足程度；
- 年度绿电缺口。

### 小时级平衡

判断海上风电出力与数据中心连续负荷之间的时间错配。

输出包括：

- 无储能时小时级绿电匹配率；
- 不同储能时长下的匹配率提升；
- 达到 80%、90%、95%、100% 小时级无碳匹配所需储能容量；
- 连续低风时段下的剩余零碳缺口；
- 需要通过绿电交易、长时储能、蓄冷、算力调度或稳定零碳电源补足的电量。

## M7. 全球外推方法

中国作为详细案例后，可将方法外推到全球沿海地区。

外推逻辑：

1. 根据数据中心增长潜力识别高需求沿海国家和城市；
2. 根据海水温度和自然冷却小时数识别高冷却节能潜力地区；
3. 根据海上风电资源识别高零碳供电潜力地区；
4. 根据电力市场、海域政策和并网条件识别可实施性；
5. 将全球沿海地区划分为若干政策类型。
